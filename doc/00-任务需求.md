# 任务需求（MVP 0.1.0）

## 背景与目标

在 Microsoft Word 中以 **AsciiMath** 的方式书写与编辑数学公式，并尽量以 **Word 原生公式（OMML）** 的形式插入到文档中，便于后续在 Word 内继续编辑/排版。

本仓库目标是实现一个 **Word Add-in（Office.js 任务窗格插件）**：

- 在任务窗格中输入 AsciiMath
- 实时预览公式（任务窗格内）
- 一键插入到文档（插入为 Word 原生公式）
- 选中已插入的公式后可读取并更新

## 关键需求

- 技术栈：TypeScript + Vue3（模块化、现代化工程结构）
- 转换链路：AsciiMath → LaTeX → MathML → OMML → 插入 Word
- 编辑体验：可“更新”已插入公式（通过选择内容控件并替换其内容）
- 代码要求：简单、鲁棒、可扩展；错误可见、不会静默失败
- 文档：目前只维护中文文档，放在 `doc/`

## 非目标（当前版本刻意不做）

- 不做复杂的语法高亮编辑器（先用 textarea）
- 不做离线 sideload 自动化/签名发布（先面向本机开发调试）
- 不实现“只选中公式一部分也能识别”的复杂定位（先以内容控件为边界）
- 不实现云端 API 调用（转换全部在本地完成；仅依赖 Office.js CDN）

## 里程碑建议

1. 0.1.0：可插入/更新 OMML + 基础预览
2. 0.2.x：更好的选区识别、公式列表管理、批量更新
3. 0.3.x：更强的编辑器（快捷输入、模板、快捷键）、导入导出
