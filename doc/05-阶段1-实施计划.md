# 阶段 1（Word 桌面端）：稳住 OMML → Word 原生公式

目标：在 Windows Word 桌面端，保证 “AsciiMath 一键更新” 的公式插入/更新稳定、可预期、可回滚。

## 交付清单（DoD）

- 插入的对象是 **Word 原生公式**（可在 Word 里用公式工具编辑/复制）
- “插入/更新” 在常见场景稳定：
  - 光标在普通段落
  - 光标在表格单元格
  - 选中已插入的 WordMath 公式后更新
- “从选中公式加载” 能正确读取 AsciiMath（包含复制/粘贴后的公式）
- OOXML 生成遵循 `insertOoxml` 推荐形态（Flat OPC / `pkg:package`）

## 技术要点与工作项

### 1) OOXML 生成规范化

- 使用 Flat OPC 包：
  - `/[Content_Types].xml`
  - `/_rels/.rels`
  - `/word/_rels/document.xml.rels`（可为空）
  - `/word/document.xml`
- 将 OMML 放在 `<w:p>` 里：
  - display 公式：`<m:oMathPara> ... </m:oMathPara>`
  - 后续若支持行内：改为 `<w:r><m:oMath>...</m:oMath></w:r>`

当前实现位置：`src/core/ooxml.ts`

### 2) 插入/更新的边界策略

- 统一用 `ContentControl(tag=wordmath)` 作为“可管理公式”的边界
- 更新只替换控件内容（`RangeLocation.content`）而不替换控件本身

当前实现位置：`src/core/word.ts`

### 3) 源码持久化与复制粘贴

- 将 AsciiMath 源码写入隐藏文本：`WMASCII:<base64-utf8>`
- 读取时从 `getOoxml()` 的结果中解析 `WMASCII:`

当前实现位置：`src/core/ooxml.ts`、`src/core/word.ts`

### 4) 桌面端验证矩阵（建议你执行，我配合修）

由于本环境无法直接启动 Word UI 做自动化验证，建议你按下面用例手工验证并把异常现象/截图发我：

- 插入后是否显示为公式对象（不是普通文本）
- Word 内“转换为线性格式/专业格式”是否可用
- 更新是否会丢失格式（字体大小、对齐、所在段落样式）
- 复制/粘贴到：
  - 同一文档不同位置
  - 新建文档
  是否还能 “从选中公式加载”
